# divelog
zxuファイルのデータをデータベース化し、web上に表示するためのプロジェクト。そろそろダイコン買い替えになりそうなので。

## workflow フォルダ
下記処理を自動化するための処理コード。コスト削減と開発優先のため、まずはローカルで実施することを想定としています。そのうちcosmosdb+change feed + functionに移行したいな。

- [前提] zxuファイルを特定フォルダ(zxu)に配置
- zxuファイルをパースし、json化
- json化したデータを特定フォルダ(json)に保存
  - ファイル名： <yyyyMMddHHmmss>-<その日の本数>-<ダイブサイト名>.json


## web フォルダ
divelogデータをweb上に表示するための処理コード。コスト削減と開発優先のため、まずはローカルで実施することを想定しています。そのうち webapp+cosmosdb に移行したいな。

### 技術stack
- Python 3.13
- Streamlit

### 表示機能
- ログ検索機能
  - json ディレクトリ内のjsonファイル名を一覧表示（名前降順)
- ログ表示機能
  - 選択されたログの詳細表示
    - ダイブサイト
      - ポイント名
      - ポイント緯度経度マッピング
    - ダイブ日時
    - 最大深度
    - 平均深度
    - 水温
    - 潜水時間
      - エントリ時間
      - エグジット時間
      - 潜水時間合計
    - 水面休息時間
    - バディ情報
    - 使用タンク情報
    - グラフ表示（深度-時間、水温-時間）
    - ギアセットの情報
    - ダイビングメモ
- ダイビングヒートマップ
  - ポイントごとのダイブ本数を地図上にヒートマップとして表示
